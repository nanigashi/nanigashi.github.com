BMC SLR02 を買ってみた。

Posted on 2014-03-08

転職してました。どうも何某です。こんにちは。

ラーメン屋にでも転職しようとしたのですが、今度もまた同業です。前職よりゲーム色が強いかも。

5年くらい乗ったので、そろそろ新しい自転車でもと思いいろいろ調べたところ、かなり評判がいい模様。

思い切って買ってみたけど凄いねこれ。

前に乗っていた Basso viper 2010 はクロモリフレーム。初めて乗った時は、自転車ってこんなに速いのかと感動しました。

が、しかし、フルカーボンフレームの戦闘力たるや。凄いっす。

一応カーボンフレームのエントリーモデルなんですかねこれ。だとするとこれより上は想像もつかないですね。

最初乗った時は、踏んだら想像以上に進むので、身体が置いてかれる感覚に。

曲がる時は重心をかけた瞬間に反応するので焦ります。

クロモリが柔らかいって聞いてましたが、実際乗り比べるとクロモリの反応の遅さってのがわかります。クロモリは踏み込んでもグニャっとしてから力が伝わる感じ。SLR02は踏んだら即伝わる。

ただSLR01乗りの人に言わせると、SLR02は反応が遅いらしい。どんだけだよSLR01。

viperの時は100kmも走ると身体が痛くなったんですが、SLR02にしてから身体が痛くなることがほとんどなくなりました。僕の身体にあってるみたいです。
坂も非常に楽に登れるので最近は坂に目覚めつつあります。

乗るのが凄く楽しくなる自転車。本当に買ってよかった。おすすめです。

zsh の correct が必要ないのに毎回出てしまう時の対応

Posted on 2014-01-31

便利な correct がうざい時に対応したメモ。

環境

  • OS: Ubuntu 12.04 64bit
  • zsh: zsh 4.3.17 (x86_64-unknown-linux-gnu)

現象

zsh で正しく打っているのにもかかわらず、間違った correct が毎回表示されてしまうコマンドがある。

次の例では、 apptitude と正しく打っているが .aptitude ではないかと表示される。

対応内容

nocorrect の後ろに実行したいコマンドを付けると correct されないのでエイリアスを作成する。

sudo が追記されているのは sudo 経由の場合、correct が復活してしまうから。なので、sudo 自体も nocorrect とする。

これで毎回 n を押すことがなくなりました。
